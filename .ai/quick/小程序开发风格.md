# 微信小程序分层编排编码风格

页面 JS 是数据流编排器，生命周期方法只做调度，数据获取/处理/展示三层分离，WXML 保持声明式渲染，WXSS 统一设计系统。

## 使用场景

- 编写新页面（JS + WXML + WXSS）
- Review 小程序前端代码是否符合团队风格
- 重构已有页面为分层编排风格
- 新成员入职需要了解前端编码规范

## 分层职责规范

每一层只做自己该做的事，职责不得混淆：

| 层级 | 职责 | 禁止事项 |
|------|------|---------|
| **app.js** | 全局生命周期、登录流程、全局数据管理 | 不写页面业务逻辑，不直接操作页面 UI |
| **Page JS** | 页面级数据编排：生命周期调度、事件处理、调用 API 工具类 | 不写 HTTP 请求细节，不写复杂数据转换算法 |
| **WXML** | 声明式模板渲染：数据绑定、条件渲染、列表渲染、事件绑定 | 不写复杂表达式逻辑，复杂计算用 WXS 模块 |
| **WXSS** | 样式与视觉表现：布局、颜色、动画、响应式适配 | 不用内联样式，不写 JS 动态样式（除非必须） |
| **Utils（api.js）** | API 调用封装：静态方法映射后端接口 | 不处理 UI，不调用 `this.setData()` |
| **Utils（request.js）** | HTTP 底层封装：请求发送、响应拦截、错误统一处理 | 不写业务逻辑，不含特定接口路径 |
| **Utils（loading.js）** | 加载状态管理：引用计数式 show/hide 配对 | 不写业务逻辑 |
| **WXS 模块** | 模板内数据转换：格式化、过滤、简单计算 | 不调用 wx API，不发网络请求 |

### 独立抽取原则

以下逻辑需**从 Page JS 中抽取**，避免页面膨胀：

#### 1. API 调用统一到 api.js

所有后端请求通过 `API` 类的静态方法调用，Page JS 中不直接使用 `request()`：

```javascript
// ✅ 正确：通过 API 工具类调用
const API = require('../../utils/api')
const res = await API.getTransactions(params)

// ❌ 错误：页面中直接调用 request
const { request } = require('../../utils/request')
const res = await request('/api/transaction/list', 'GET', params)
```

#### 2. 复杂数据转换抽到独立函数

当数据处理逻辑超过 5 行时，抽取为页面内的独立方法或工具函数：

```javascript
// ✅ 正确：数据处理抽为独立方法
formatTransactionGroups(transactions) {
  const groups = {}
  transactions.forEach(item => {
    const date = item.date
    if (!groups[date]) groups[date] = { date, items: [], totalExpense: 0, totalIncome: 0 }
    groups[date].items.push(item)
    if (item.type === 0) groups[date].totalExpense += parseFloat(item.amount)
    else groups[date].totalIncome += parseFloat(item.amount)
  })
  return Object.values(groups).sort((a, b) => b.date.localeCompare(a.date))
},

// ❌ 错误：在 fetchData 中堆砌 20 行数据转换
```

#### 3. WXML 复杂计算用 WXS 模块

模板中需要的格式化、映射、计算逻辑放入 WXS 模块：

```xml
<!-- ✅ 正确：复杂逻辑用 WXS -->
<wxs module="fmt">
  var formatAmount = function(amount) {
    return parseFloat(amount).toFixed(2)
  }
  module.exports = { formatAmount: formatAmount }
</wxs>
<text>{{fmt.formatAmount(item.amount)}}</text>

<!-- ❌ 错误：模板中写复杂三元表达式 -->
<text>{{item.amount > 0 ? '+' + (Math.round(item.amount * 100) / 100).toFixed(2) : ...}}</text>
```

## 核心原则

1. **生命周期即调度器**：`onLoad`、`onShow` 只做方法调用编排，不写具体逻辑
2. **setData 最小化**：只更新变化的字段，避免整体替换大对象，减少渲染开销
3. **async/await + try-catch-finally**：所有异步操作统一使用 async/await，配合 try-catch-finally 处理加载态
4. **Loading 配对管理**：使用 `Loading` 工具类的引用计数机制，确保 show/hide 配对
5. **登录态回调模式**：页面通过 `app.onLoginReady(callback)` 等待登录完成，不轮询不猜测
6. **事件委托 + dataset**：列表项事件通过 `data-*` 属性传参，在事件处理中从 `e.currentTarget.dataset` 获取
7. **防重复操作**：按钮点击通过 `_submitting` 标志位防止重复提交
8. **缓存优先**：用户信息等不常变数据优先从 `wx.getStorageSync` 读取，减少网络请求

## 操作步骤

### 编写新页面

1. 创建页面四件套：`index.js`、`index.wxml`、`index.wxss`、`index.json`
2. 在 `index.js` 中定义 `data` 初始状态，包含所有模板依赖的字段
3. 在 `onLoad` / `onShow` 中编排数据加载流程
4. 每个数据获取方法遵循 `async + try-catch-finally` 模式
5. 事件处理方法按功能分组，命名统一用 `on` 或 `handle` 前缀
6. 在 `index.wxml` 中使用声明式绑定，复杂逻辑抽到 WXS
7. 在 `index.wxss` 中使用 rpx 单位，遵循设计系统配色

### 添加新 API 调用

1. 在 `utils/api.js` 中添加静态方法
2. 在页面 JS 中通过 `API.methodName()` 调用
3. 不直接在页面中使用 `request()`

## 输出格式

### Page JS 结构模板

编写的页面 JS **MUST** 符合以下结构：

```javascript
const app = getApp()
const API = require('../../utils/api')
const Loading = require('../../utils/loading')

Page({
  /**
   * 页面数据
   */
  data: {
    loading: true,
    listData: [],
    // ... 所有模板依赖的字段都在此声明初始值
  },

  /**
   * 生命周期 —— 只做调度
   */
  onLoad(options) {
    app.onLoginReady((authorized) => {
      if (!authorized) {
        wx.reLaunch({ url: '/pages/login/index' })
        return
      }
      this.initPage(options)
    })
  },

  onShow() {
    if (app.globalData.isAuthorized === true) {
      this.fetchData()
    }
  },

  onPullDownRefresh() {
    this.fetchData().then(() => wx.stopPullDownRefresh())
  },

  /**
   * 数据获取 —— async + try-catch-finally
   */
  async fetchData() {
    this.setData({ loading: true })
    try {
      // 步骤1：获取列表数据
      const res = await API.getList(this.data.params)

      // 步骤2：格式化数据
      const formattedList = this.formatListData(res.data)

      // 步骤3：更新视图（最小化 setData）
      this.setData({
        listData: formattedList,
        loading: false
      })
    } catch (err) {
      console.error('fetchData failed:', err)
      Loading.error(err.message || '加载失败')
      this.setData({ loading: false })
    }
  },

  /**
   * 数据转换 —— 独立方法
   */
  formatListData(list) {
    // 超过 5 行的转换逻辑独立成方法
    return list.map(item => ({
      ...item,
      displayAmount: parseFloat(item.amount).toFixed(2),
      displayDate: item.date.substring(5) // MM-DD
    }))
  },

  /**
   * 事件处理 —— on/handle 前缀
   */
  onItemTap(e) {
    const { id } = e.currentTarget.dataset
    wx.navigateTo({ url: `/pages/detail/index?id=${id}` })
  },

  async handleSubmit(e) {
    // 防重复提交
    if (this._submitting) return
    this._submitting = true

    Loading.show('提交中...')
    try {
      const formData = e.detail.value
      await API.createItem(formData)
      Loading.success('提交成功')
      wx.navigateBack()
    } catch (err) {
      Loading.error(err.message || '提交失败')
    } finally {
      Loading.hide()
      this._submitting = false
    }
  }
})
```

### WXML 结构模板

```xml
<!-- 页面容器 -->
<view class="container">

  <!-- 加载状态 -->
  <view wx:if="{{loading}}" class="loading-wrap">
    <text>加载中...</text>
  </view>

  <!-- 主内容 -->
  <view wx:else class="content">

    <!-- 列表渲染：必须指定 wx:key -->
    <block wx:for="{{listData}}" wx:key="id">
      <view class="list-item" data-id="{{item.id}}" bindtap="onItemTap">
        <text class="item-title">{{item.name}}</text>
        <text class="item-amount {{item.type === 0 ? 'expense' : 'income'}}">
          {{item.type === 0 ? '-' : '+'}}{{item.displayAmount}}
        </text>
      </view>
    </block>

    <!-- 空状态 -->
    <view wx:if="{{listData.length === 0}}" class="empty-state">
      <text>暂无数据</text>
    </view>

  </view>
</view>
```

### API 工具类模板

```javascript
// utils/api.js
const { request } = require('./request')

class API {
  // 每个方法对应一个后端接口，静态方法 + 清晰命名
  static async getList(params) {
    return request('/api/resource/list', 'GET', params)
  }

  static async createItem(data) {
    return request('/api/resource/create', 'POST', data)
  }

  static async deleteItem(id) {
    return request('/api/resource/delete', 'POST', { id })
  }
}

module.exports = API
```

## 自检清单

编写完成后逐条检查：

**分层职责**
- [ ] 页面 JS 是否只做数据编排，不含 HTTP 请求细节？
- [ ] API 调用是否全部通过 `utils/api.js` 的静态方法？
- [ ] 复杂数据转换是否抽为独立方法（超过 5 行）？
- [ ] WXML 模板是否只做声明式渲染，复杂计算用 WXS 模块？
- [ ] WXSS 是否使用统一配色和 rpx 单位，无内联样式？

**生命周期编排**
- [ ] `onLoad` / `onShow` 是否只做方法调度，无具体逻辑？
- [ ] 是否通过 `app.onLoginReady()` 等待登录完成？
- [ ] `onPullDownRefresh` 是否调用 `wx.stopPullDownRefresh()`？

**数据管理**
- [ ] `data` 中是否声明了所有模板依赖的字段初始值？
- [ ] `setData` 是否只更新变化的字段（最小化更新）？
- [ ] 列表是否指定了 `wx:key`？

**异步与状态**
- [ ] 所有异步操作是否使用 `async/await + try-catch-finally`？
- [ ] Loading 是否使用 `Loading` 工具类配对管理？
- [ ] 表单提交是否有防重复标志位 `_submitting`？

**样式规范**
- [ ] 是否使用设计系统颜色（#07c160 / #333 / #666 / #999 / #f7f8fa）？
- [ ] 金额数字是否使用 `'DIN Alternate'` 字体？
- [ ] 卡片是否使用 `width: 92%` + `border-radius: 20rpx` 统一风格？
- [ ] 是否使用 Flexbox / Grid 布局，无浮动？
- [ ] 动画是否使用 CSS `@keyframes`，时长 0.3-0.5s？

**命名规范**
- [ ] JS 变量/方法是否为 camelCase？
- [ ] CSS 类名是否为 kebab-case？
- [ ] 事件处理方法是否使用 `on` / `handle` 前缀？
- [ ] 数据获取方法是否使用 `fetch` / `load` 前缀？

## 注意事项

- 此风格适用于**页面级编排**，不适用于工具类内部实现
- 工具类方法内部可以有条件分支和循环，但页面生命周期方法 MUST NOT
- 当页面逻辑过于复杂（超过 300 行）时，考虑拆分为自定义组件
- WXS 模块不支持 ES6 语法，只能使用 `var`、`function` 等 ES5 语法
- `setData` 不要传递超过 1MB 的数据，大列表考虑分页或虚拟列表
- 页面间传参优先使用 URL query，复杂对象用全局变量或 EventChannel
